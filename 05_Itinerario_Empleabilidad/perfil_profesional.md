## Quién soy

Estudio el Ciclo Superior de Desarrollo de Aplicaciones Web en Prometeo by The Power, modalidad virtual. Es mi primer año programando. Antes de empezar no sabía qué era un `JOIN` ni había escrito una sola línea de Java. Ahora tengo un proyecto que integra una base de datos relacional, una aplicación con arquitectura en capas y una web responsive con diseño propio.

## Qué tipo de proyectos me gustan

Me interesan los proyectos que tienen lógica de negocio real detrás. No me basta con que algo funcione visualmente: quiero entender por qué está estructurado así, qué problema resuelve y cómo se conectan las piezas. El proyecto intermodular me ha enseñado que diseñar bien la base de datos antes de escribir una sola línea de código es más importante que dominar la sintaxis.

Me llama especialmente el desarrollo backend y la arquitectura de aplicaciones. Entender cómo fluyen los datos desde la base de datos hasta lo que ve el usuario es lo que más me ha enganchado este año.

---

## En qué me gustaría especializarme

En desarrollo backend, probablemente con Java o algún lenguaje moderno orientado a ese perfil. Me interesa aprender a construir APIs REST el año que viene, que es lo que conectaría la aplicación Java con la web en un proyecto como NexHub. También quiero profundizar en el diseño de bases de datos y en testing, que es algo que este año no he podido trabajar tanto como me hubiera gustado.

---

## Qué me motiva

Ver algo funcionar que yo he construido desde cero. Cuando ejecuté por primera vez la aplicación Java, se conectó a MySQL y devolvió los datos de los socios en consola, eso fue lo que me confirmó que quería seguir en esto. No fue espectacular, pero era mío y funcionaba.

También me motiva la idea de que el software resuelve problemas reales. NexHub no es solo un ejercicio de clase: está modelado como si fuera un encargo real, con una empresa ficticia que tiene nombre, dirección y precios concretos.

## Lo que mejoraría

Conectar la web con la aplicación Java mediante una API REST, porque ahora mismo son dos piezas independientes que representan lo mismo pero no se comunican. Y añadir tests unitarios a los servicios con JUnit, que es algo que aprendí tarde que debería haber hecho desde el principio.
